## Commands

All commands run from `src/`:

```bash
# Development
NODE_OPTIONS=--openssl-legacy-provider npm run dev       # dev server on :3000
NODE_OPTIONS=--openssl-legacy-provider npm run build     # build + export → src/out/ + build/

# Linting
npm run lint          # ESLint with auto-fix across pages/, components/, lib/, layouts/, scripts/

# No test suite — verify visually via dev server or inspect src/out/index.html after build
```

The `NODE_OPTIONS=--openssl-legacy-provider` flag is required due to the Node.js / OpenSSL version mismatch with Next.js 11.

`postbuild` copies `src/out/` → `build/` and `build/yuweichen1008/` for GitHub Pages deployment.

## Architecture

### Repo layout

```
src/          Next.js app (the only place to edit)
build/        Auto-generated by postbuild — never edit manually
scripts/      Repo-level git hooks (pre-push)
```

### Data sources

Pages pull from two sources, merged at build time in `getStaticProps`:

1. **Notion API** — `src/lib/notion.js` — food log, adventures, timeline events, life chapters, daily stats. Requires env vars (see `.env.local`). All Notion functions return `[]` / `null` gracefully when `NOTION_TOKEN` is absent.
2. **Local markdown** — `src/data/journal/*.md` — journal entries with frontmatter (`title`, `date`, `slug`, `categories`, `mood`, `summary`, `location`). Parsed by `src/lib/localJournal.js`. Locale variants use `.zh.md` / `.ja.md` suffixes alongside the primary `.md`.

**The landing page (`pages/index.js`) uses only local data** — `getStaticProps` calls `getLocalJournalEntries()` directly and falls back to `src/data/nowData.js`. No Notion token needed to build the home page.

### i18n

Three locales: `en` / `zh` / `ja`. Translation strings live in `src/locales/[locale].json`. Access via:

```js
const { t, locale } = useTranslation()  // from @/lib/i18n
```

`LocaleProvider` (in `_app.js`) wraps the whole app. Locale is persisted to `localStorage` and auto-detected from `navigator.language`. All three locale files must be updated together when adding new keys.

### Routing

Static export with a manual `exportPathMap` in `next.config.js`. **Adding a new page requires registering it in `exportPathMap`**, or it won't appear in the exported `out/` directory. Dynamic routes (journal slugs, timeline slugs) are enumerated from Notion + local data at build time within that map.

### Tailwind

JIT mode enabled. `src/tailwind.config.js` purges from `pages/**`, `components/**`, `layouts/**`. When writing new utility classes in data constants (e.g. color strings in `SKILL_ENTRIES` in `index.js`), the full class string must appear literally — no dynamic concatenation — or it will be purged.

### Key components

- `LayoutWrapper` — header + footer shell; all pages render inside it
- `TrainerCard` — Pokemon-style hero card on the landing page; type badges + stat bars
- `WorldMap` — Fog of World travel map; loaded with `next/dynamic({ ssr: false })` because react-simple-maps uses D3 which breaks SSR. Visited countries come from `src/data/fowData.js` as ISO 3166-1 numeric codes matching TopoJSON IDs.
- `NotionRenderer` — renders Notion block JSON to JSX (paragraphs, headings, lists, callouts, images)
- `CalendarGrid` — 35 KB component; month-view activity log with milestone highlighting and step sparklines
- `StepsWidget` / `NowSection` — live step tracking + "Right now" items (Notion-backed with static fallback)
- `CategoryConfig` (`src/lib/categoryConfig.js`) — single source of truth for category emoji, dot colors, and pill colors used across journal, calendar, and filter components

### Notion env vars

Each Notion database has its own ID env var. The token and all DB IDs must be set for Notion-backed pages to show live data. Without them, pages either show empty state or fall back to static data files in `src/data/`. See `.env.local` for the full list of keys.

### Production React → Preact swap

`next.config.js` replaces `react` / `react-dom` with `preact/compat` in production client bundles only. Dev server and SSR use real React. Avoid `ReactDOM.createPortal` and other APIs not in Preact compat.
